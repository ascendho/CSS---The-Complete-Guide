Question 1

What's a "Rule" in CSS?

1
2
3

Correct Answer

2

A combination of selector ("WHAT to style") and declaration ("HOW to style it").

Explanation

This is correct, the rule includes both the WHAT and HOW.

Question 2

Which selector would select this element?

<h1 class="page-title">Dive into the core features</h1> 

1
2
3
4

Correct Answer

4

.page-title { ... } 

Explanation

This is correct, the dot at the beginning is required to select elements with a class of "page-title".

Question 3

What is "Specificity" about?

1
2
3

Correct Answer

3

Specificity is all about resolving conflicts that arise from multiple CSS rules which target the same element.

Explanation

This is correct. If two or more rules target the same element and set different styles for the same property, this conflict needs to be resolved.

Question 4

What's a Combinator?

1
2
3

Correct Answer

1

A CSS feature which allows you to combine two selectors in a dependent way.

Explanation

This is correct because combinators (~, +, >, ) tie two selectors together (in a way defined by the combinator).

Question 5

Why should you NOT use inline styles (<div style="..."> )?

1
2
3

Correct Answer

3

Because the code becomes harder to maintain and predict.

Explanation

If you use inline styles, it becomes hard to see which styles are added to which element for which reason. And you end up with a lot of (unreadable) style declarations in HTML. In bigger HTML files, this becomes hard to read and work with.
